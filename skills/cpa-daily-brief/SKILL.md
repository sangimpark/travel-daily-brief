# CPA Daily Brief 📊🔥

CPA 마케팅 블로거를 위한 매일 아침 1장 요약 도우미.
"리서치 페이즈"를 줄여 사용자가 **판단·선택**에만 집중하도록 한다.

이 스킬은 parksangim83의 OS 인터뷰 첫 결과물 (2026-05-07).
OS 선언문: "나는 판단·선택만 하고, 나머지는 시스템이 한다."

---

## When to use this skill

자동 발동 트리거:
- "오늘 추천", "오늘 글감", "오늘 뭐 쓰지"
- "CPA 추천", "어제 성과"
- "/오늘추천", "/cpa브리프"

발동하지 말 것:
- 일반적인 블로그 글쓰기 도움 (이건 다른 스킬)
- 비-CPA 마케팅 콘텐츠 (여행 후기 등 — 같은 블로그라도 다른 트랙)
- 단순 잡담이나 무관한 "추천" 키워드 (예: "어제 추천받은 책이…")

## How this skill works

이 스킬은 4단계로 흐른다 (총 3~5분).

### Step 1 · 어제 성과 받기

**목적**: 어제 발행한 글이 있으면 성과 데이터를 수집.

**핵심 질문**:
> "어제 발행한 글이 있나요? 있으면 — URL 또는 제목 + CPA 플랫폼별 클릭/전환/수익을 알려주세요. 없으면 '없음'."

**발판** (사용자가 막힐 때):
> "예: '쿠팡파트너스 - 5월 신규 카테고리 정리, 47클릭/3전환/12,400원'"

**스킵 조건**:
- 사용자가 "어제 발행 없음" 또는 "기억 안 남"이라고 답하면 → 어제 단계 건너뛰고 Step 2로.

### Step 2 · 광고주 사이트 점검

**목적**: 등록된 광고주/CPA 플랫폼의 현재 상태를 **모두 자동으로** 점검.

**핵심 동작** (sources.json 있을 때):
1. `data/sources.json` 읽기.
2. **세 묶음으로 처리**:

   **A. `sources` 배열 (WebFetch)** — auto_fetch:true 항목
   - 각 URL을 WebFetch로 병렬 점검
   - `fetch_prompt_hint` 필드를 prompt로 사용해 사이트별 맞춤 추출
   - 등록: 트립닷컴 / 마이리얼트립 / 트래블로카 (모바일·해외 호텔·항공·패키지)

   **B. `chrome_fetch` 배열 (Claude in Chrome MCP)** — 봇 차단·로그인 SPA 우회용
   - **사전 셋업** (한 사이클에 1회만):
     1. `mcp__Claude_in_Chrome__list_connected_browsers` → 연결된 브라우저 확인
     2. `mcp__Claude_in_Chrome__select_browser` → 1대 선택
     3. `mcp__Claude_in_Chrome__tabs_context_mcp` (createIfEmpty: true) → MCP 탭 그룹 생성
   - **점검 (priority 순)**:
     - priority:1 (3hoursahead) 먼저 → priority:2 (클룩/KKday) → priority:3 (여기어때)
     - 각 항목: `browser_batch`로 [navigate, get_page_text] 묶어 호출
     - `requires_login:true`인 항목이 빈 페이지(notifications region만) 반환하면 사용자에게:
       > "{{name}} 로그인 세션이 만료된 것 같아요. 브라우저에서 로그인 후 다시 발동해주시면 자동 점검됩니다. 이번 사이클은 그 사이트 스킵하고 진행할게요."
     - `extract_method` 필드 ("get_page_text" 또는 "read_page") 사용. `fallback_extract` 명시되어 있으면 1차 실패 시 보조 사용
     - 각 사이트의 `watch_for` / `notes` 컨텍스트로 답변 정리에 반영

   **C. `manual_check` 배열** — A/B 둘 다 불가능한 사이트만
   - 현재 비어 있음 (2026-05-10부터). 새 광고주 등록 시에만 사용
   - 있으면 사용자에게 직접 입력 요청 (`user_workflow` 필드 안내)

3. **sources.json 없을 때 (첫 사용)**:
   > "오늘 점검할 광고주 사이트 URL 5개 정도 알려주세요. 다음부터는 자동으로 점검합니다."
   - 받은 URL 리스트로 `data/sources.json` 생성
   - WebFetch 한 번 시도 → 성공: `sources`로 / 실패(봇 차단): `chrome_fetch`로 자동 분류
   - 로그인 필요 사이트는 `chrome_fetch` + `requires_login: true`

**파일 형식 참고**: 현재 `data/sources.json` (2026-05-10 갱신 — chrome_fetch 카테고리 추가됨)

### Step 3 · 사용자 컨텍스트

**목적**: 오늘 글쓰기에 쓸 수 있는 시간/에너지 파악 (추천 정렬에 반영).

**핵심 질문**:
> "오늘 글쓰기에 쓸 수 있는 시간(짧음 30분 / 보통 1시간 / 충분 2시간+)과 에너지(피곤/보통/좋음)는?"

**발판** (사용자가 막힐 때):
> "지금 시간대(아침/점심/저녁)랑 컨디션(피곤/보통/좋음)만 알려주셔도 됩니다."

**활용**:
- 시간 짧음 + 에너지 피곤 → 짧은 글로 쓸 수 있는 후보 우선 (단순 후기, 할인 소식 등)
- 시간 충분 + 에너지 좋음 → 길게 쓸 가치 있는 후보 우선 (비교 분석, 심층 후기)

### Step 4 · 1장 요약 출력 + 누적 저장

**목적**: 어제 성과 + 오늘 후보 + 사용자 컨텍스트를 종합한 1장 요약 생성. **동시에 누적 폴더에 저장** (브리프 누적 — 향후 웹사이트 빌드용).

**처리 순서**:
1. **`_exclusions` 필터 적용** (가장 먼저): Step 2에서 모은 모든 후보 캠페인을 `sources.json`의 `_exclusions` 배열로 필터링. `pattern` 정규식이 `match_field`(`title` 또는 `type_or_title`)에 매칭되면 후보에서 제거. 제거된 항목은 메모로도 노출하지 않음.
2. **`_sort_order` 적용**: `sources.json`의 `_sort_order` 룰로 정렬.
   - `type_priority`에 따라 종류별 그룹 정렬 (현재: **단독 상품 > 할인**).
   - 같은 그룹 내에서는 `tiebreaker` 순 (마감 임박 → 단가 큼 → 발행 블로그 카테고리 매칭 → 시즌성).
3. Step 3 컨텍스트 기반으로 "오늘의 한 가지" 추천 1개 선정 (정렬 1순위 우선, 단 사용자 컨텍스트 강하게 매칭되면 그것).
4. 아래 Output format에 맞춰 마크다운 1장 출력 (전체 리스트 + 추천 표시).
5. **누적 저장** (신규): 같은 마크다운 내용을 `~/.claude/skills/cpa-daily-brief/data/briefs/{YYYY-MM-DD}.md` 파일로 동시 저장.
   - 같은 날짜 파일이 이미 있으면 덮어쓰기 (하루에 한 번 발동 가정).
   - 파일 헤더에 frontmatter (YAML)로 메타데이터 추가:
     ```yaml
     ---
     date: 2026-05-10
     blog_target: kkakkung_mom  # Step 0/3에서 받은 발행 블로그
     selected: "#1 KKday 싱가포르 디즈니 크루즈"  # "오늘의 한 가지" 그대로
     total_candidates: 8  # _exclusions 후 표시 캠페인 수
     ---
     ```
   - 누적된 파일들은 향후 2단계(정적 사이트) 빌드에 자료원으로 사용.

## Harness rules

### 규칙 1 · 추상 답 차단

**언제 발동**: 사용자가 "그냥 다", "아무거나", "아무 거나 추천해줘" 같은 회피로 답함.

**무엇을 한다**:
> "지금 가장 먼저 떠오르는 광고주/플랫폼 1개만요. 거기서 풀어볼게요."

### 규칙 2 · 데이터 부족 차단

**언제 발동**: Step 1에서 어제 성과를 "기억 안 남"으로 답하거나, Step 2에서 사이트 등록을 못 함.

**무엇을 한다**:
> "그러면 그 단계 건너뛰고 가능한 부분만 정리할게요. 다음 발동 때 다시 시도하시면 됩니다."

### 규칙 3 · 시간대 발판

**언제 발동**: Step 3에서 사용자가 "잘 모르겠다"로 답함.

**무엇을 한다**:
> "지금 시간대(아침/점심/저녁) + 컨디션(피곤/보통/좋음)만 알려주셔도 됩니다."

### 규칙 4 · Chrome MCP 미연결 차단

**언제 발동**: Step 2의 chrome_fetch 처리 단계에서 `list_connected_browsers` 결과가 빈 배열 (Chrome 익스텐션 미연결 또는 미설치).

**무엇을 한다**:
> "Claude in Chrome 브라우저가 연결 안 되어 있어요. 익스텐션 설치 + Chrome 실행 후 다시 발동해주시면 chrome_fetch 4사도 자동 점검됩니다. 이번 사이클은 sources (WebFetch 3사)만 점검하고 진행할게요."
- 그 사이클은 chrome_fetch 항목들을 manual_check 모드로 대체 (사용자에게 짧게 직접 입력 요청)
- `manual_check`로 임시 분류된 사이트는 다음 발동에서 다시 chrome_fetch 시도

### 규칙 5 · 로그인 세션 만료 차단

**언제 발동**: chrome_fetch 중 `requires_login:true` 사이트(3hoursahead)가 빈 페이지(notifications region만) 반환.

**무엇을 한다**:
> "{{사이트명}} 로그인 세션이 만료된 것 같아요. 브라우저에서 로그인 후 다시 발동해주시면 자동 점검됩니다. 이번 사이클은 그 사이트만 스킵하고 나머지 진행할게요."

## Output format

1장 마크다운 — 정확히 다음 양식으로 출력:

```
# 📅 오늘의 CPA 브리프 — {{YYYY-MM-DD}}

## 📊 어제 발행 글 성과
- {{글 제목 또는 URL}} → 클릭 X, 전환 Y, 수익 Z원
- (어제 발행 없으면 "어제 발행 없음")

## 🔥 오늘의 프로모션 전체 리스트 ({{N}}개)
> 자료원: WebFetch 3사 + Chrome MCP 4사. `_exclusions` 자동 적용 후 표시.

| # | 광고주 | 캠페인 | 종류 | 핵심 → 추천 이유 | 마감 | 🔗 |
|:--:|---|---|---|---|---|:--:|
| 1 | {{광고주명}} | {{캠페인명}} | {{할인/단독 상품}} | {{할인율·금액·혜택 1~2단어}}`<br>`{{⭐⭐⭐ 1순위 — 이유 1줄}} | {{~M/D}} | [🔗]({{광고주 URL}}) |
| 2 | ... | ... | ... | ... | ... | ... |
| ... | | | | | | |

(표 아래) `📌 **3hoursahead 종합 페이지**: [🔗 3hoursahead 프로모션 목록](https://3hoursahead.com/ko/studio/promotions)` 한 줄 추가.

**컬럼 순서 고정**: 광고주 > 캠페인 > 종류 > 핵심 및 추천 이유 > 마감 > 🔗

**원칙**:
- `_exclusions` 매칭된 캠페인은 표에서 완전 제거 (메모도 노출 X)
- AI가 미리 5개로 거르지 말 것 — 사용자가 직접 보고 선택할 수 있게 진행 중 캠페인을 가능한 전부 나열 (보통 10~15개)
- 같은 캠페인이 두 자료원(예: 3hoursahead + 광고주 직접)에 동시 등장하면 1행으로 통합
- **광고주 / 캠페인 컬럼 분리**: 광고주명만 별도 컬럼 (검색·필터 쉽게). 캠페인명은 그 옆.
- **마감 컬럼**: 형식 `~M/D` (D-N 표기 제외, 사용자 선호). 마감 임박 시각화 안 함.
- **핵심 + 추천 이유 한 셀 — 줄바꿈으로 분리**: 형식 `{{핵심 내용}}<br>{{추천 표시 + 이유}}`
  - 1줄: 핵심 내용 (할인율·금액·혜택)
  - 2줄: 추천 표시 + 이유 (⭐ 표시로 추천임을 명확히 — 화살표 → 불필요)
  - 예: `최대 **87% 특가**<br>**⭐⭐ 2순위** — kkakkung_mom 매칭 + 마감 임박`
  - 예: `호텔+항공권<br>**⭐⭐⭐ 1순위** — 단독 상품 + 단가 큼 + 가족여행 매칭`
  - 추천 없는 항목은 1줄 핵심만 (예: `쿠알라룸푸르/발리`)
  - 핵심도 추천도 없으면 `—`
- 핵심에 들어가는 것: 할인율·금액·혜택만 1~2단어 (예: "최대 87% 특가", "10만원 즉시 할인", "호텔+항공권")
- **🔗 컬럼**: 광고주 URL 링크 (형식 `[🔗]({{URL}})`). URL은 `sources.json`의 광고주별 `url` 또는 `fetch_url` 필드 사용. sources.json에 없는 광고주(라쿠텐트래블·스테이앤모어 등)는 3hoursahead detail URL 또는 광고주명 검색 URL.
- 1순위 행은 광고주 컬럼도 **굵게**, 1·2순위 행은 핵심 부분(할인율·금액)도 **굵게**
- 표 아래 "광고주 자체 사이트에서 추가 발견" 보조 리스트 1~3줄 (마감일 모호하거나 상시 행사)

## 🎯 오늘의 한 가지 (시간 X분 / 에너지 Y 기준)
→ **#{{N}} {{이름}}**: {{이유 1~2줄, 사용자 컨텍스트·발행 블로그 카테고리와 연결}}.
단, 사용자 판단으로 표의 다른 항목 선택 가능 ({{대안 1~2개 짧게 언급}}).

## 💡 메모
- 다음에 점검할 후보: ...
- 등록된 광고주 리스트: data/sources.json (N개)
```
